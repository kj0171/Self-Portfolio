4) BLACKBUCK | DISTRIBUTED TRACING

Distributed tracing is basically a monitoring tool for the distributed microservice architecture. My role in this project was to do learn and implement distributed tracing for the blackbuck infrastructure. A zipkin server was hosted ,applications based on Node JS, a custom middleware was introduced which reported data to the zipkin server and for the java applications, spring sleuth was introduced which is wrapper for brave library used to report data to the zipkin server. Every request was tagged with a unique request id and same was logged. A developer could look into the Kibana for the logs with a crisp request id to trace the complete flow of the request. Elastic search was used as the storage for the requests with the retention period marked as 7 days. The other advantage of distributed tracing is one can run a spark job and form the service dependency graph which can help in detecting circular dependency in the system.

Technology : Zipkin, Node js, Spring sleuth, Java, Elastic Search, Spark
